# Interview Growth Focus

Three pillars to develop intentionally.

---

## 1. Algorithm - LeetCode 75

**Goal:** Build reliable pattern recognition and clear implementation under time pressure.

| Area | Focus |
|------|-------|
| Arrays & Strings | Two pointers, sliding window, prefix/suffix reasoning |
| Hashing | Frequency maps, complements, grouping |
| Trees & Graphs | Traversal, recursion state, BFS/DFS tradeoffs |
| Dynamic Programming | State definition, transition, base cases |
| Heaps & Intervals | Ordering, scheduling, top-k choices |

**Growth signal:** Can you explain the invariant before writing code?

---

## 2. System Design - System Design Interview Book

**Goal:** Move from naming components to reasoning about scale, reliability, tradeoffs, and operations.

| Area | Focus |
|------|-------|
| Requirements | Functional scope, non-functional targets, APIs |
| Data & Storage | Schema, indexing, partitioning, consistency |
| Scale | Caching, load balancing, sharding, queues |
| Reliability | Backpressure, retries, failover, observability |
| Tradeoffs | Latency vs consistency, cost vs durability, simplicity vs extensibility |

**Growth signal:** Can you defend one design under changing constraints?

---

## 3. Behavioral Interview - Common Questions

**Goal:** Tell specific, senior-signal stories with ownership, judgment, and measurable outcomes.

| Area | Focus |
|------|-------|
| Story Bank | Conflict, failure, leadership, ambiguity, impact |
| Structure | STAR, situation pruning, action ownership |
| Seniority Signal | Tradeoffs, influence, scope, durable follow-through |
| Delivery | Concise opening, crisp transitions, metric-backed result |
| Reflection | Learning, changed behavior, team-level impact |

**Growth signal:** Does the answer prove judgment, not just effort?

---

## Weekly Practice

- **2 algorithm sessions** - one new pattern or one timed LeetCode 75 problem.
- **1 system design session** - one prompt with requirements, API, data model, architecture, and bottleneck review.
- **1 behavioral session** - one common question answered aloud and tightened.
